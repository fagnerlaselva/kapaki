  <action>
1. Start the production build and preview server:
```bash
npm run build && npx nuxi preview
```

2. Run Lighthouse CLI against the local preview (install if needed: `npm install -g lighthouse`):
```bash
lighthouse http://localhost:3000 --output=json --output-path=./lighthouse-report.json --chrome-flags="--headless --no-sandbox" --preset=perf --form-factor=mobile
```

If `lighthouse` CLI is not available or fails (headless Chrome not available in this environment), use `npx unlighthouse` or fall back to manual verification in the checkpoint task.

3. Parse the JSON report for Performance and SEO scores.

4. **If Performance < 90 (mobile)**, investigate and fix common causes:
   - Check if nuxt-gtag adds render-blocking scripts — if so, configure `loadingStrategy: 'async'` in nuxt.config.ts gtag block
   - Check for excessive `<link rel="prefetch">` tags — if too many, consider adding `nuxt-vitalizer` module
   - Check LCP element — hero image must have `loading="eager"` and `fetchpriority="high"` (already done in Phase 2/3, verify)
   - Check CLS — font loading with fallback metrics (already configured via @nuxt/fonts)

5. **If SEO < 100**, investigate:
   - Missing meta description (already configured in all pages)
   - Missing viewport meta (already in nuxt.config via htmlAttrs)
   - robots.txt blocking crawlers
   - Missing canonical URL

6. After fixes, rebuild and re-run Lighthouse to confirm scores.

7. Also verify: run `npm run dev` and check browser console for any hydration mismatch warnings or gtag errors.

NOTE: If Lighthouse CLI cannot run in this environment (no Chrome), document the scores as "requires manual verification" and the checkpoint task below will handle it.
