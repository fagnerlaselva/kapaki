**2. `app/components/ServicosSection.vue`** — Section wrapper with 3 cards:

Props via defineProps:
```typescript
interface ServiceCardData {
  key: string
  title: string
  description: string
  ctaLabel: string
  badge?: string
}

defineProps<{
  services: ServiceCardData[]
  waAssistencia: string
  waAcessorios: string
  waAparelhos: string
}>()
```

Template structure:
- Root: `<section id="servicos" class="bg-surface py-16 px-4">`
- CRITICAL: `id="servicos"` is required for the HeroSection anchor link `#servicos` to work
- Inner container: `<div class="max-w-6xl mx-auto">`
- Title: `<h2 class="font-heading text-3xl font-bold text-text text-center mb-4">O que voce precisa hoje?</h2>`
- Subtitle: `<p class="text-text-muted text-center mb-12">Escolha seu caminho — a Kapaki resolve.</p>`
- Grid: `<div class="grid grid-cols-1 md:grid-cols-3 gap-8">`
- Loop over services with v-for, mapping WhatsApp URLs per service key:

```vue
<ServiceCard
  v-for="service in services"
  :key="service.key"
  :title="service.title"
  :description="service.description"
  :cta-label="service.ctaLabel"
  :badge="service.badge"
  :featured="service.key === 'assistencia'"
  :whatsapp-url="service.key === 'assistencia' ? waAssistencia : service.key === 'acessorios' ? waAcessorios : waAparelhos"
/>
```

This ensures each card gets its DISTINCT WhatsApp URL (WA-02 requirement). The page will call useWhatsApp('assistencia'), useWhatsApp('acessorios'), useWhatsApp('aparelhos') and pass each URL separately.

CRITICAL: Do NOT use a single generic whatsappUrl for all cards. Each service MUST have its own pre-filled message.

**2. Update `app/layouts/default.vue`** — Mount AppFooter globally:

The layout calls `useAppConfig()` to assemble footer data. This is the ONE exception to "no composables in layout" — the layout layer is an orchestrator above page layer.

Also call `useWhatsApp('geral')` for the footer's WhatsApp link.

```vue
<script setup lang="ts">
const config = useAppConfig()
const { whatsappUrl: footerWaUrl } = useWhatsApp('geral')
</script>

<template>
  <div class="min-h-screen bg-background text-text font-body">
    <slot />
    <AppFooter
      :site-name="(config.siteName as string)"
      :address="(config.address as { street: string; city: string; state: string; zip: string })"
      :phone="(config.phone as string)"
      :whatsapp-url="footerWaUrl"
      :instagram="(config.instagram as string)"
      :hours="{ weekday: 'Seg-Sex 9h-18h', saturday: 'Sab 9h-13h', sunday: 'Fechado' }"
    />
  </div>
</template>
```

Note on type casting: `useAppConfig()` returns `AppConfig` with potentially untyped fields. Cast with `as string` or inline type to avoid template type errors.
